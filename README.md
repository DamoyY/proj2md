# proj2md

proj2md 是一个简单的命令行工具，用于将整个代码项目的目录结构和文件内容提取并合并为一个单一的 Markdown 文件。

该工具适合在需要将项目代码上下文整体提供给大语言模型时使用。

## 主要功能

* **生成目录树**：在文档开头清晰地展示项目的目录结构。
* **合并文件内容**：将项目中的源代码文件内容提取到文档中，并根据文件扩展名自动添加 Markdown 代码块语法高亮标记。
* **自动过滤**：
  * 自动读取并遵守项目中的 `.gitignore` 规则。
  * 默认忽略 `README.md`、`LICENSE` 以及上一次生成的 `project.md` 等非代码文件。
  * 自动检测并跳过二进制文件（在文档中会标记为“二进制文件”而不会输出乱码）。
* **编码兼容**：支持读取 UTF-8（含 BOM）编码的文件，并在可能的情况下自动识别和处理其他编码格式。
* **一键复制**：运行结束后，自动将生成的 Markdown 文件复制到系统剪贴板（当前仅支持 Windows 操作系统）。

## 安装

确保你的系统已安装 Rust 编译环境。克隆本仓库后，在项目根目录下运行以下命令进行编译：

```sh
cargo build --release
```

编译完成后，可执行文件将位于 `target/release/proj2md.exe`（Windows）。你可以将其移动到系统的环境变量 `PATH` 包含的目录中，以便在任何地方全局使用。

## 使用方法

proj2md 是一个纯命令行工具，使用非常简单。它最多接受一个路径参数。

### 1. 处理当前目录

如果不提供任何参数，proj2md 将以当前命令行所在的目录作为项目根目录进行处理：

```sh
proj2md
```

### 2. 处理指定目录

你也可以显式传入一个目标项目的路径（绝对路径或相对路径均可）：

```sh
proj2md /path/to/your/project
```

### 运行结果

运行成功后，终端会输出类似如下的提示：

```text
正在生成文档...
文档文件已复制到剪贴板: C:\Users\Username\AppData\Local\Temp\proj2md\project.md
```

此时，包含整个项目结构的 Markdown 文件不仅已经保存在了系统的临时目录中，并且该**文件本身**已经复制到了你的系统剪贴板中。你可以直接在支持粘贴文件的聊天窗口（如各种 AI 助手的对话框）中使用 `Ctrl+V` 粘贴。

## 产物格式说明

生成的 `project.md` 文件将包含两个主要部分：

### 1. 目录结构
以缩进的方式展示遍历到的所有有效文件和文件夹。

```markdown
## 1. 目录结构

your-project/
    src/
        main.rs
        utils.rs
    Cargo.toml
```

### 2. 文件内容
逐一展示文件的相对路径和具体代码内容。

````markdown
## 2. 文件内容

### src/main.rs
```rs
fn main() {
    println!("Hello, world!");
}
```

### Cargo.toml
```toml
[package]
name = "your-project"
version = "0.1.0"
```
````

## 平台支持说明

由于工具集成了操作系统的剪贴板文件拖拽（File Drop）功能，目前 **仅完全支持 Windows 操作系统**。如果在非 Windows 平台上运行，文件生成逻辑可以正常工作，但在尝试复制文件到剪贴板时会提示不支持的错误。
